# P2-A Late Fusion — 설계 근거, 결과, 한계

> P1(ECG)에 IMU·SpO2를 결합하는 멀티모달 융합의 첫 접근(late fusion)을 정리한다. 설계 선택의
> 근거, 실험 결과, 그리고 이 접근이 도달한 한계와 그 함의를 다룬다. 수치 상세는 `records/` 참조.

---

## 개요

동시 측정 멀티모달 데이터가 부재한 제약 위에서, 클래스 조건부 조립으로 합성 paired 데이터를 구성하고
gated late fusion을 설계했다. 데이터 누수 점검을 포함한 검증을 거쳐 정직한 분리 성능(macro-F1
0.94)을 확보했으나, 분석 결과 **조건부 독립으로 조립한 합성 데이터에서는 교차모달 상호작용을 학습할
수 없다**는 구조적 한계를 확인했다. 이 한계가 후속 두 방향 — 실 정렬 데이터 기반 cross-modal
attention(P2-B)과 실데이터 검증 기반 시스템 통합 — 의 동기가 되었다.

---

## 1. 배경 — 동시 측정 멀티모달 데이터의 부재

목표는 P1(ECG 심장분석)의 출력에 IMU·SpO2를 결합해 응급 원인을 5분류(정상 / 운동 / 심혈관 / 낙상 /
저산소)하는 것이다. 제약은 데이터에 있었다. ECG·IMU·SpO2를 같은 피험자에게서 동시에 측정하고 응급
라벨을 부여한 공개 데이터셋이 없다(후보 데이터셋은 모달리티가 분리돼 있거나 SpO2가 결여).

멀티모달 모델은 모달리티별 인코더(이종 데이터로 독립 학습 가능)와 융합 헤드(동시 데이터 필요)로
분리된다. 동시 데이터의 부재를 다루기 위해, **클래스 조건부 독립 가정**(클래스가 주어지면 각
모달리티의 분포가 근사적으로 독립) 아래 각 모달리티를 자기 클래스 분포에서 표집해 합성 paired
샘플을 구성했다(방법 A). 이 가정의 타당 범위가 본 연구의 핵심 한계로 이어진다(5절).

ECG 인코더는 별도 학습하지 않고 P1(ECG-FM + LoRA, 768차원 임베딩)의 출력을 직접 사용했다. CPSC
데이터에 대한 P1 임베딩을 산출해 캐시했다.

---

## 2. 데이터 전략과 전처리

합성 사전분포를 SisFall(낙상)·PTT-PPG(운동)의 실측 IMU 피처로 보정하는 과정에서 두 정합성 문제를
교정했다.

- **단위·스케일 불일치**: 일부 클래스는 실측, 일부는 문헌 추정값이라 `jerk_peak`가 클래스 간 수백
  배 차이.
- **샘플링·윈도우 불일치**: 데이터셋 간 fs(500/200Hz)·윈도우(전체기록/3초)가 달라 피처가 비교 불가.

모든 IMU를 200Hz·3초 윈도우로 통일하고 동일 조건 실측으로 재보정해 해소했다. 성능 저하의 원인을
모델에 귀속하기 전에 데이터 파이프라인의 정합성부터 검토해야 함을 보여주는 사례다.

---

## 3. 융합 아키텍처 비교

세 방식을 비교했다.

- **Concat MLP**(베이스라인): 모달리티 결측에 취약(SpO2 제거 시 macro-F1 0.95→0.79).
- **학습형 게이팅**: 세 변형 모두 입력과 무관한 고정 가중치로 수렴(게이트 붕괴). 합성 데이터에서는
  고정 혼합비가 손실을 최소화하는 해라, 입력 의존 게이팅을 학습할 유인이 없다.
- **신뢰도 기반 라우팅(conf-routed, 채택)**: 확신도가 가장 높은 expert에 가중을 두는 무학습 규칙.
  학습 파라미터가 없어 붕괴가 발생하지 않으며, 클래스별 라우팅이 명확히 분리(심혈관→ECG, 낙상→IMU,
  저산소→SpO2). test macro-F1 0.95.

이 비교는 합성 데이터에서 학습형이 무학습 규칙보다 불리할 수 있음을 보인다 — 데이터에 학습 가능한
교차모달 구조가 없으면, 자유도가 큰 모델은 가장 단순한 해로 수렴한다. 이 관찰이 cross-modal
attention의 타당성 재검토(5절)로 이어진다.

---

## 4. 검증 — 데이터 누수 점검과 성능 보정

초기 융합 성능(test F1 0.96)이 비정상적으로 높아 누수를 점검했고, 세 결함을 교정했다.

1. **ECG 임베딩 누수**: 캐시 복원추출 후 분할해 test 임베딩의 90.5%가 train과 동일. → P1
   train/test 풀을 분리해 누수 제거.
2. **Bootstrap 근접 누수**: 실벡터의 5% 지터 재사용으로 test가 train의 근접 복제. → 공분산 보존
   다변량 가우시안(MVN) 표집으로 교정.
3. **고차원 임베딩 과적합**: 누수 제거 후 val 0.96 / test 0.86(격차 0.10). → 16차원 병목으로
   격차 0.007로 축소.

보정 후 성능은 macro-F1 **0.94**다. 이 점검 절차 자체가 방법론적 신뢰의 근거가 된다.

## 5. 한계와 함의

핵심 질문은 "이 합성 융합을 cross-modal attention 등으로 더 정교화할 가치가 있는가"였다. 두 분석이
부정적 결론을 가리켰다.

- ECG-FM의 312개 시간 토큰이 상호 cos 0.999로 거의 동일 — 멀티토큰 시간 어텐션의 근거가 약하다.
- 더 본질적으로, 합성 데이터가 조건부 독립 조립이라 ECG와 IMU가 서로 다른 피험자·시점에서 유래한다.
  **동일 시점의 "ECG 이상 ↔ IMU 충격" 같은 시간적 교차모달 대응이 데이터에 존재하지 않는다.**

따라서 본 접근의 한계는 분명하다. **조건부 독립으로 조립한 합성 데이터에서 융합이 학습할 수 있는
것은 정적 피처 공간의 분리와 오경보 억제까지이며, 교차모달 상호작용은 학습 대상이 될 수 없다.**
후자는 시간 정렬된 실측 멀티모달 데이터를 요구한다.

함의는 두 방향의 후속 연구로 정리된다.

- **교차모달 상호작용 학습**에는 실 정렬 데이터가 필요하다 → cross-modal attention을 별도 트랙
  (P2-B, `WidU_P2_B_Cross_Modal_Attention`)으로 분리한다. 합성 proxy로 평가 방법론(오경보율·attention
  XAI)을 먼저 확립하고, 교차모달 상호작용의 본 학습은 실 정렬 데이터 확보 후 수행한다.
- **현 시점에서 신뢰 가능한 시스템**은 합성 학습 융합이 아니라, 실데이터로 검증된 검출기와 투명한
  결합 규칙으로 구성한다 → 실데이터 검증·통합 트랙으로 이어진다.

---

## 방법론적 함의 (요약)

1. 조건부 독립으로 조립한 합성 데이터는 교차모달 시간 대응을 담지 못해, 융합 정교화의 효용이 제한된다.
2. 설계 판단은 의견이 아니라 실측으로 닫는다(풀링 효용·토큰 유사도·누수율 모두 정량 검증).
3. 데이터 누수는 상시 점검 대상이다(임베딩 90.5% 누수, bootstrap 근접 누수 사례).
4. 성능 저하의 원인을 모델에 귀속하기 전에 데이터 파이프라인의 정합성(단위·윈도우)을 확인한다.
